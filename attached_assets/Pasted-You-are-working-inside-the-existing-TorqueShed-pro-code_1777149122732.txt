1. Stripe checkout

Add backend endpoint:

Payload:
{
  "plan": "diy_pro" | "garage_pro" | "shop_pro"
}

Behavior:
- Requires authenticated user.
- Looks up the correct Stripe Price ID from env.
- Creates or reuses a Stripe customer for the current user.
- Creates a Stripe Checkout subscription session.
- Includes metadata:
  userId
  plan
  app: "torqueshed"
- success_url should return to /account/billing?checkout=success
- cancel_url should return to /pricing or /account/billing?checkout=cancelled
- Return session.url to the frontend.

Frontend behavior:
- Upgrade buttons must call this endpoint.
- Redirect user to Stripe Checkout.
- Disable button and show loading state while session is being created.
- Show clear errors if Stripe config is missing.
