You are working inside the existing TorqueShed repository. This is NOT a greenfield build. Your job is to refactor, redesign, and complete the app into a production-ready, visually premium, feature-rich automotive community platform with zero prototype rough edges.

CORE PRODUCT GOAL
TorqueShed must become the best mobile-first home base for mechanics, enthusiasts, and DIY owners to:
1. track their vehicles and builds,
2. get credible help fast,
3. discover fitment-aware parts and tools,
4. buy/sell parts with trust,
5. build identity and reputation inside vehicle-specific communities.

PRIMARY POSITIONING
“The Garage for Real People”
But the actual product experience must feel like:
- part build journal,
- part community bay,
- part diagnostic copilot,
- part trusted swap marketplace.

NON-NEGOTIABLE OUTCOMES
- Make the app visually stunning and cohesive across mobile and web.
- Remove all shallow prototype behavior and incomplete UX.
- Eliminate architecture drift, duplicate navigation patterns, dead code, placeholder flows, and inconsistent data sources.
- Preserve and improve the current Expo + React Native + Express + Drizzle + Postgres stack unless a change is absolutely necessary.
- Use strict TypeScript. No unsafe shortcuts, no silent failures, no hidden regressions.
- The result must feel premium, trustworthy, and production-ready.

PRODUCT RESTRUCTURE
Restructure the app around these pillars:

1. HOME
- personalized feed based on user vehicles, joined bays, followed builds, saved searches, and recent activity
- sections such as:
  - For Your Garage
  - Solved This Week
  - Trending Builds
  - Parts Deals
  - Local Swap Picks
- show credibility and freshness signals on all content

2. BAYS
- brand/platform-specific communities
- content tabs inside each bay:
  - Questions
  - Builds
  - Solved
  - Deals
  - Local
- each bay needs strong identity, hero header, member stats, top contributors, recent solved issues
- support search, filters, pinned content, and “ask question” CTA

3. MY GARAGE
- transform vehicle notes into full build journals
- each vehicle should support:
  - overview
  - maintenance log
  - repair history
  - mod list
  - issue tracker
  - photo gallery
  - parts used
  - future plans
  - cost summary
- allow private/public sharing options for build entries
- public build pages should be compelling and community-friendly

4. SOLVE FLOW
- create a structured “Fix My Issue” flow:
  - vehicle
  - symptoms
  - codes
  - recent changes
  - severity
  - drivability
  - AI-assisted triage
- AI should help structure the problem, not replace community trust
- threads should support accepted solution, tools used, parts used, estimated difficulty, estimated cost, and repair summary
- solved content must become reusable knowledge, not buried conversation

5. SOURCE
- merge parts finder and marketplace thinking into a stronger discovery experience
- parts finder must feel fitment-aware even if some vendor integrations remain link-based
- show:
  - best matches
  - fitment confidence
  - vendor options
  - price range
  - why it matters
  - install difficulty
  - related threads/builds
- marketplace and swap shop must include:
  - better listing cards
  - seller trust metadata
  - saved items
  - share listing
  - report listing
  - inquire/message seller
  - improved media galleries

6. PROFILE AND CREDIBILITY
- profiles must feel valuable and trustworthy
- include:
  - avatar
  - bio
  - location
  - specialties
  - years wrenching
  - vehicles owned/worked on
  - solved thread count
  - successful swaps
  - review score
  - verified owner / verified seller / shop badge states
- reputation should be visible but tasteful

- Before adding new UI, identify and remove structural duplication.

- If a screen currently exists but feels unfinished, redesign it fully instead of decorating it.
- Reuse design primitives consistently.
- Do not leave “coming soon” surfaces in core navigation.
- If third-party integrations are not available yet, build clean interfaces and fallback UX rather than fake implementations.
- Keep the app emotionally compelling, but prioritize credibility over hype.
- Make the product feel like it was designed by people who actually understand enthusiast culture, marketplace trust, and mobile UX.
